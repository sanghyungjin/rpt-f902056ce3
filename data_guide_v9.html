5.1 특허 매칭 2-Tier 구조 (v9 Phase 1, 2026-04-10 업데이트)

v9 부터 특허 분류가 NCBI BLASTp 에서 Aho-Corasick deterministic exact substring containment 으로 변경되었습니다. BLASTp 의 heuristic seed-and-extend 알고리즘은 short charged-cluster peptide (8-12 aa) 에서 silent false negative 가 있었으며, 이를 수학적으로 완전한 Aho-Corasick 으로 교체했습니다.

Tier PCPC_Status FTO_Status 의미 판정 기준
A PATENT_CLAIM_ENTITY BLOCKED 짧은 특허 entry 안에 query 가 exact substring 으로 존재 → entry 자체가 peptide claim 후보 entry_length ≤ max(30, 3 × query_length)
B PATENT_COMMON_SUBSTRING CAUTION query 가 bulk protein (수백~수천 aa) 의 내부 substring 으로 존재 → 해당 특허는 큰 construct 를 claim 하는 것이지 이 짧은 substring 자체를 claim 하지 않을 가능성 높음 Tier A 에 안 걸린 나머지 exact hit
Domain partition: query length 3-5 aa 는 특허 exact check 대상이 아닙니다. pataa (769M residues) 에서 random 5-mer 의 기대 hit 수가 ~240 이므로 "모든 5-mer 가 어딘가에 substring 으로 존재"하는 noise floor 입니다. 3-5 aa 는 내부 cosmetic/bioactive DB 검사만 적용됩니다. 6 aa 부터 entry-length 기반 Tier A/B 분류가 의미 있는 selectivity 를 가집니다 (Tier A BLOCKED 비율: 6mer 52%, 10mer 29%).

8. Conservation (컬럼 27-28)

# 컬럼명 타입 설명 가능한 값
27 Avg_Conservation float window 내 평균 per-residue conservation (Human DSP 와 2 lizard query 의 alignment-based score) 0.0-1.0. v9 dual threshold filter (avg ≥ 0.9) 통과
28 Min_Conservation float (v9 신규) window 내 최소 per-residue conservation (가장 보존도가 낮은 한 residue) 0.0-1.0. v9 dual threshold filter (min ≥ 0.5) 통과
Dual conservation threshold (avg ≥ 0.9 AND min ≥ 0.5): single threshold (avg only) 로는 "평균은 높지만 특정 position 이 완전 unconserved" 인 peptide 가 통과하는 false-positive 문제가 발생합니다. v9 는 두 조건을 동시에 요구하여 entire window 의 conservation quality 를 보장합니다.

Conservation map 자체는 BioPython PairwiseAligner (global mode, BLOSUM62, open_gap=−10, extend_gap=−0.5) 로 재구축되었습니다. 이전 시리즈의 naive position-wise zip() 방식 (frame shift artifact 발생) 을 폐기한 결과입니다.
